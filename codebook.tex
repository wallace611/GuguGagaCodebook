\subsection{SOS DP}
目標是解決求給定集合的所有子集和：
$$F(S)=\sum_{x\subseteq S}A(x)$$
直接窮舉子集複雜度為 $O(3^n)$，使用 SOS DP 則可優化至 $O(n \cdot 2^n)$。

\textbf{高維前綴和概念 (High-Dimensional Prefix Sum)} \\
SOS DP 的本質其實就是高維度的前綴和推廣：
\begin{enumerate}
    \item \textbf{一維前綴和}：沿著單一方向累加，\texttt{F[x] += F[x-1]}。
    \item \textbf{二維前綴和}：先固定 $Y$ 對 $X$ 做一維前綴和 (\texttt{F[x][y] += F[x-1][y]})，再固定 $X$ 對 $Y$ 做一維前綴和 (\texttt{F[x][y] += F[x][y-1]})。幾何上就是把一條條的前綴和（線）疊加成一個矩形（面）。
    \item \textbf{N 維前綴和 (SOS DP)}：在 bitmask 中，共有 $n$ 個維度，每個維度的大小只有 0 和 1。我們只需依序窮舉每一維 $i$，若當前狀態 $s$ 在該維度是 1 (即 \texttt{s \& (1 << i)} 為真)，就將長度較短的狀態（即該維度為 0 的狀態 $s \oplus (1 \ll i)$）累加到 $F[s]$ 中。
\end{enumerate}

\subsection{Counting DP}
用於計算方法數的 DP，通常可以配合排列組合與排容原理。

\textbf{障礙物棋盤走路計數問題} \\
給定 $n \times m$ 棋盤與 $N$ 個障礙物，只能向下或右走，求起點到終點且不經過障礙物的方法數。
利用總和扣除不合法路徑的方法：
$$dp[D] = cnt(O,D) - \sum_{\{k\}} cnt(k,D) \cdot dp[k]$$
轉移順序：將點先按 $X$ 排序，再按 $Y$ 排序，確保 partial-ordering 正確。

\subsection{Probabilistic DP}
\subsection{Probabilistic DP (機率與期望值 DP)}
核心為利用 DP 狀態轉移模擬全機率公式與期望值的線性性質。

\subsubsection*{一、 兩大轉移方向}
\begin{itemize}
    \item \textbf{求機率（通常順推 Forward）：} 
    定義 $dp[i]$ 為從起點抵達狀態 $i$ 的機率。初始 $dp[start] = 1$。\\
    轉移方程：$dp[next] \mathrel{+}= dp[cur] \times P(cur \to next)$
    \item \textbf{求期望值（通常逆推 Backward）：} 
    定義 $E[i]$ 為從狀態 $i$ 至終點的期望步數或花費。初始 $E[end] = 0$。\\
    轉移方程：$E[cur] = \sum P(cur \to next) \times (E[next] + cost)$\\
    \textit{優勢：} 逆推可自動過濾無法抵達終點的無效狀態，確保計算收斂。
\end{itemize}

\subsubsection*{二、 經典模型與狀態設計}
\begin{itemize}
    \item \textbf{擲骰子模型（古典機率與背包）：} 
    處理獨立事件的多次操作（屬有向無環圖 DAG 轉移）。\\
    \textit{精度細節：} 迴圈內避免頻繁除法。應\textbf{先累加前置狀態的總和，最後再統一除以分支數}。
    \item \textbf{抓老鼠模型（雙方博弈與條件機率）：} 
    處理輪流操作且目標隨機減少的問題。轉移時需窮舉當前所有可能事件分支（例如：主角直接贏、對手抽中干擾且引發不同的後續變化）。\\
    \textit{實作細節：} 必須明確定義極端邊界條件（如只有目標物機率為 $1$，只有干擾物機率為 $0$）。
\end{itemize}

\subsubsection*{三、 實戰 Checklist}
\begin{itemize}
    \item \textbf{圖中是否有環？ (Cyclic Transitions)} \\
    若狀態轉移成環（如有一定機率停留原地），常規 DP 會死循環。此時需將轉移式視為線性聯立方程式，建立矩陣並使用\textbf{高斯消去法 (Gaussian Elimination)} 求解。
    \item \textbf{浮點數型別：} \\
    只要牽涉機率與期望值，一律使用 \texttt{double} 或 \texttt{long double}。
    \item \textbf{期望值的線性性質：} \\
    利用 $E[A + B] = E[A] + E[B]$。處理多個獨立物件時，無需將其壓縮成單一高維狀態，可分拆計算各自期望值後相加，大幅降低時間複雜度。
\end{itemize}

\subsection{Bitmask DP}
位元壓縮 DP，泛指狀態是由二進制表示的 DP。通常可以表示集合，0 表示不取，1 表示取。假設有 $n$ 個東西要考慮，使用長度為 $n$ 的 bitstring 表示狀態，複雜度通常為 $O(2^n)$。看到題目範圍 $n \le 20$ 即可考慮此解法。

\textbf{搭電梯問題 (Elevator Problem)} \\
$n$ 個人排隊，每個人重 $w_i$，要搭一台負重 $x$ 的電梯，問最少要搭幾趟？利用 $O(2^n)$ 窮舉最後上電梯的人，減少了窮舉 $n!$ 排列的時間複雜度。
